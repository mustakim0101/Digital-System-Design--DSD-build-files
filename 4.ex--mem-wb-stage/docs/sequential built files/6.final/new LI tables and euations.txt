# 1) Opcode = `000`  → **ALU / R-type**

**Purpose:** do one of your ALU operations (16 ops)

* **ALU selector (funct) = D4–D1** ✅ goes to S3..S0 **when ALUOp indicates R-type**
* **D0 = don’t care** (you can keep 0 or 1)

## ALU Control Logic (S3..S0 generation)

We keep your ALU-control style based on **ALUOp** and **Func bits**.

### Define:

* **Func3 = D4**
* **Func2 = D3**
* **Func1 = D2**
* **Func0 = D1**

### ALU_Control_3 (Most Significant Bit)

* It is `1` only when **ALUOp=10** AND **Func3=1**
* **Equation:**
  `ALU_Control_3 = (ALUOp1 AND NOT(ALUOp0) AND Func3)`

### ALU_Control_2

* **Equation:**
  `ALU_Control_2 = (ALUOp1 AND NOT(ALUOp0) AND Func2)`

### ALU_Control_1

* **Equation:**
  `ALU_Control_1 = (ALUOp1 AND NOT(ALUOp0) AND Func1)`

### ALU_Control_0 (Least Significant Bit)

It is `1` in two cases:

1. When **ALUOp=01** (forced SUB for BEQ)
2. When **ALUOp=10** AND **Func0=1** (R-type funct)

* **Equation:**
  `ALU_Control_0 = (NOT(ALUOp1) AND ALUOp0) OR (ALUOp1 AND NOT(ALUOp0) AND Func0)`

✅ In this opcode `000` format:

* **ALU selector = D4 D3 D2 D1**
* **There is NO rs/rt/rd encoded**
* Inputs come from your **fixed or mux-selected registers** (RA/RB forced logic)
* **offset = not present**

# 2) Opcode = `001`  → **LW (load word)**

**Purpose:** address = Reg[rs] + offset, then load memory data

* **rs (base register) = D4–D3**
* **offset (signed 3-bit) = D2–D0** ✅ goes into sign extender
* **rt is not encoded separately**
* **destination (write register) = RT bits = D2–D1** (as you requested for WA)

So for opcode `001`:

* **rs = D4 D3**
* **offset = D2 D1 D0**
* **write destination WA = D2 D1 (RT field you’re using)**
* **ALU selector is forced to ADD (0000)** (because LW uses ALUOp=00)

---

# 3) Opcode = `010`  → **SW (store word)**

**Purpose:** address = Reg[rs] + offset, then store data to memory

* **rs (base register) = D4–D3**
* **offset (signed 3-bit) = D2–D0**
* **no register write**

So for opcode `010`:

* **rs = D4 D3**
* **offset = D2 D1 D0**
* **WA unused** (RegWrite = 0)
* **ALU selector forced to ADD (0000)** (because SW uses ALUOp=00)

---

# 4) Opcode = `011`  → **BEQ (branch equal)**

**Purpose:** compare two registers (Reg[rs] vs Reg[rt]) using SUB

* **rs = D4–D3**
* **rt = D2–D1**
* **D0 don’t care**

So for opcode `011`:

* **rs = D4 D3**
* **rt = D2 D1**
* **ALU selector forced to SUB (0001)** (because BEQ uses ALUOp=01)

---

# 5) Opcode = `100` → **LI (Load Immediate)** ✅ NEW

**Purpose:** load a small constant into a register
This gives you MOV-like behavior for small numbers (0..7 or signed −4..+3).

## Instruction Format

* **RD (destination register) = D4–D3**
* **Immediate (signed 3-bit) = D2–D0**
* **ALU will compute: 0 + Immediate**
* WB writes the ALU result into RD

So for opcode `100`:

* **rd = D4 D3**
* **imm = D2 D1 D0**
* **WA = RD = D4 D3** ✅ (write destination comes from RD field)
* **ALUSrc = 1** (choose sign-extended immediate for B input)
* **ALU selector forced to ADD (0000)** (we want 0 + imm)
* **MemtoReg = 0** (write ALU result back)
* **MemRead = 0, MemWrite = 0**

### Important datapath note for LI

Since LI requires `0 + imm`, you must ensure ALU input A is **0** (either by forcing RA to a register that holds 0, or by adding a mux that selects constant 0 into ALU_A when LI=1).

# Fixed / mux-selected registers summary (by opcode)

### ✅ Opcode `000` = ALU / R-type

Because you used **D4–D1 as FUNCT**, you do not have rs/rt bits here, so you forced fixed registers:

* **RA = 00 → A always reads R0**
* **RB = 01 → B always reads R1**
* **WA = 00 → destination always R0** (since LW=0 and LI=0, WA mux outputs 00)

Fixed registers used:

* **A input fixed = R0**
* **B input fixed = R1**
* **Destination fixed = R0**

---

### ✅ Opcode `001` = LW (load)

LW uses instruction fields:

* **RA = RS = IR4..IR3** (base register)
* **RB = fixed 01 (R1)** but ignored in ALU input because **ALUSrc=1** selects offset for B
* **WA = RT = IR2..IR1** (destination register)

Fixed registers used:

* **B register selection stays fixed to R1**, but it’s not used for ALU because ALUSrc=1
* Destination comes from RT field

---

### ✅ Opcode `010` = SW (store)

SW uses RS for address base:

* **RA = RS = IR4..IR3** (base register)
* **RB = fixed 01 (R1)** and normally this is the store-data source
* **WA don’t care** (RegWrite=0)

Fixed registers used:

* **Store-data register fixed = R1** (since RB fixed when not BEQ)

*(If later you want store data from RT bits, RB must select IR2..IR1 during SW too.)*

---

### ✅ Opcode `011` = BEQ

BEQ compares two registers:

* **RA = RS = IR4..IR3**
* **RB = RT = IR2..IR1** (because BEQ=1 forces RB mux to instruction bits)
* **WA don’t care** (RegWrite=0)

Fixed registers used:

* **None** (both chosen from instruction bits)

LI writes a constant into RD:

* **RA behavior:** not important if you force ALU_A = 0; otherwise RA selects whatever feeds A.
* **RB:** ignored for ALU input because **ALUSrc=1** selects the immediate.
* **WA = RD = IR4..IR3** (destination register)
* ALU performs **0 + imm**, WB writes result to RD

## Where WA1/WA0 come from

* **Instruction provides candidates:**

  * **RT bits = IR2..IR1**
* **Fixed default candidate:**

  * **00** (R0)

Your **WA 2-bit MUX** chooses between these.

### WA MUX rule (what we set)

* **If LW=1 → WA = RT = IR2..IR1**
* **If LW=0 → WA = 00 (fixed)**

So:

* **WA0 = output of mux channel selecting IR1 vs 0**
* **WA1 = output of mux channel selecting IR2 vs 0**

Those WA outputs then go into the **74LS139 write decoder** inputs.

## When is WA “default” vs “fixed”?

* **Default = “from instruction” (RT)** happens only for **LW** (opcode `001`) because only LW writes a register using your encoding.
* **Fixed = 00** happens when **LW=0**, which includes:

  * **ALU opcode `000`** (R-type): RegWrite=1, but you don’t have Rd bits → we used **fixed destination R0**
  * **SW (`010`) and BEQ (`011`)**: RegWrite=0, so write decoder is disabled anyway (WA doesn’t matter, but it will still sit at 00 because LW=0)

## Do you get all 4 combinations (00,01,10,11) on WA?

✅ **Yes, but only during LW instructions.**
Because in LW format you encoded **RT = IR2..IR1**, so RT can be:

* 00, 01, 10, 11

So during LW:

* **WA = IR2..IR1 = {00,01,10,11}** ✅

During ALU instructions (opcode 000):

* **WA is fixed to 00** (because LW=0 and we chose fixed dest=R0)
