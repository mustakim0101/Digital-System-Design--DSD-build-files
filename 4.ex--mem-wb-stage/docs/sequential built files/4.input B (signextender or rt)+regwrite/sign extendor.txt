Got it — you want a **sign extender**, not zero extender, and you want the **exact pin-by-pin wiring** for:

[
\textbf{ALU_B[7:0]} = \text{MUX}(\text{ReadDataB[7:0]}, \text{SignExt[7:0]}, \text{select}=\text{ALUSrc})

We’ll still use **two 74LS157** for the 8-bit mux. The only change is how we build **SignExt[7:0]** from IR2..IR0.

# A) Sign extension for a 3-bit offset (IR2..IR0)

### What “sign extension” means here

* Your offset is 3 bits: **IR2 IR1 IR0**
* **IR2 is the sign bit**

  * IR2=0 → positive (0..+3)
  * IR2=1 → negative (−4..−1) in 2’s complement 3-bit form

So sign extension to 8 bits means:

* **SignExt0 = IR0**
* **SignExt1 = IR1**
* **SignExt2 = IR2**
* **SignExt3 = IR2**
* **SignExt4 = IR2**
* **SignExt5 = IR2**
* **SignExt6 = IR2**
* **SignExt7 = IR2**

✅ So unlike zero-extend, the upper five bits are **copies of IR2**, not GND.

### Quick examples

* offset = `011` (+3) → SignExt = `0000 0011`
* offset = `101` (−3) → SignExt = `1111 1101` ✅

No IC needed: it’s just wiring.

# B) 8-bit 2:1 mux for ALU_B using two 74LS157

We will do:

* **A inputs = ReadDataB**
* **B inputs = SignExt**
* **SEL = ALUSrc**
* **/G = 0 (enabled)**

74LS157 behavior:

* SEL=0 → output = A inputs
* SEL=1 → output = B inputs

So:

* **ALUSrc=0 → ALU_B = ReadDataB**
* **ALUSrc=1 → ALU_B = SignExt(offset)**
