## Fixed registers summary (by opcode)

### вң… Opcode `000` = ALU / R-type

Because you used **D4вҖ“D1 as FUNCT**, you **do not have rs/rt bits** here.
So you forced fixed registers:

* **RA = 00 вҶ’ A always reads R0**
* **RB = 01 вҶ’ B always reads R1**
* **WA = 00 вҶ’ destination always R0** (since LW=0, WA mux outputs 00)

 Fixed registers used:

* **A input fixed = R0**
* **B input fixed = R1**
* **Destination fixed = R0**


### вң… Opcode `001` = LW (load)

LW uses instruction fields:

* **RA = RS = IR4..IR3** (base register)
* **RB = fixed 01 (R1)** but **ignored** because **ALUSrc=1** selects offset for B
* **WA = RT = IR2..IR1** (destination register)

 Fixed registers used:

* **B register selection stays fixed to R1**, but itвҖҷs not used because ALUSrc=1
* No fixed destination (destination is RT)

### вң… Opcode `010` = SW (store)

SW uses RS for address base:

* **RA = RS = IR4..IR3** (base register)
* **RB = fixed 01 (R1)** and **normally this is the store-data source**

  * Store data still comes from **ReadDataB** (from regfile B port), so RB matters there.
* **WA donвҖҷt care** (RegWrite=0)


 Fixed registers used:

* **Store data register is fixed = R1** (because RB fixed 01 when not BEQ)
* Destination none (no reg write)

> If later you want store data from RT, you must make RB select = IR2..IR1 for SW too (like BEQ does). Right now itвҖҷs fixed.

BEQ compares two registers:

* **RA = RS = IR4..IR3**

* **WA donвҖҷt care** (RegWrite=0)

* **None** (both are chosen from instruction bits)

## Where WA1/WA0 come from

* **Instruction provides candidates:**

  * **RT bits = IR2..IR1**
* **Fixed default candidate:**

  * **00** (R0)

Your **WA 2-bit MUX** chooses between these.

### WA MUX rule (what we set)

* **If LW=1 вҶ’ WA = RT = IR2..IR1**
* **If LW=0 вҶ’ WA = 00 (fixed)**

So:

* **WA0 = output of mux channel selecting IR1 vs 0**
* **WA1 = output of mux channel selecting IR2 vs 0**

Those WA outputs then go into the **74LS139 write decoder** inputs.

## When is WA вҖңdefaultвҖқ vs вҖңfixedвҖқ?

* **Default = вҖңfrom instructionвҖқ (RT)** happens only for **LW** (opcode `001`) because only LW writes a register using your encoding.
* **Fixed = 00** happens when **LW=0**, which includes:

  * **ALU opcode `000`** (R-type): RegWrite=1, but you donвҖҷt have Rd bits вҶ’ we used **fixed destination R0**
  * **SW (`010`) and BEQ (`011`)**: RegWrite=0, so write decoder is disabled anyway (WA doesnвҖҷt matter, but it will still sit at 00 because LW=0)

## Do you get all 4 combinations (00,01,10,11) on WA?

вң… **Yes, but only during LW instructions.**
Because in LW format you encoded **RT = IR2..IR1**, so RT can be:

* 00, 01, 10, 11

So during LW:

* **WA = IR2..IR1 = {00,01,10,11}** вң…

During ALU instructions (opcode 000):

* **WA is fixed to 00** (because LW=0 and we chose fixed dest=R0)
