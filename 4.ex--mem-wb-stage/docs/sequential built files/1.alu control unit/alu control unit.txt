Here is a detailed, gate-level plan for designing and building the ALU Control Unit.

## **🧠 DETAILED DESIGN: ALU CONTROL UNIT**

### **1. Objective and Core Logic**

The goal of the ALU Control Unit is to generate a 4-bit output signal, `ALU_Control[3:0]`, based on two inputs:
1.  The 2-bit `ALUOp` signal from the Main Control Unit (`ALUOp1`, `ALUOp0`).
2.  The 4-bit `Func` field from the instruction itself (`Func[3:0]`, which are bits `D4, D3, D2, D1`).

The logic is as follows:

| `ALUOp1` | `ALUOp0` | Operation Type | `ALU_Control[3:0]` Output | Source of Output |
|:---:|:---:|:---|:---|:---|
| `0` | `0` | R-Type / Memory Reference | `0000` | Hardcoded `ADD` |
| `0` | `1` | Branch | `0001` | Hardcoded `SUBTRACT` |
| `1` | `0` | R-Type | `Func[3:0]` | Instruction `Func` field |
| `1` | `1` | (Unused) | `XXXX` | Don't Care (can be `0000`) |

*   **Why `0000` for `ALUOp=00`?** For `LOAD` and `STORE` instructions, the ALU is used to calculate the memory address (e.g., `Register + Offset`). This is always an `ADD` operation. According to your `alu table.txt`, `ADD` corresponds to `0000`.
*   **Why `0001` for `ALUOp=01`?** For `BRANCH` instructions, the ALU must perform a `SUBTRACT` to check if `rs == rt` (i.e., `rs - rt == 0`). According to your `alu table.txt`, `SUBTRACT` corresponds to `0001`.
*   **Why `Func` for `ALUOp=10`?** For actual R-Type instructions (like `ADD`, `SUB`, `AND`, etc.), the specific operation is determined by the `Func` field. The main control unit sets `ALUOp` to `10` to tell the ALU Control, "Look at the `Func` bits and pass them through."

---

### **2. Implementation Strategy: A 4-to-1 MUX**

The most structured way to implement this logic is with a **4-to-1 Multiplexer**. The select lines of this MUX will be `ALUOp1` and `ALUOp0`. The four data inputs of the MUX will be the four possible sources for the ALU operation code.

Since a 4-to-1 MUX is not a standard 74-series IC, we will build it using **three 74LS157 (Quad 2-to-1 MUX)** ICs. Each 74LS157 can handle 2 bits of our 4-bit output, so we need two for the main 4-to-1 logic, and one to help combine them. A simpler way is to use two 74LS157s, where each one handles one 4-bit path, and then a third to select between them. However, the cleanest way is to think of it as selecting one of four 4-bit inputs.

Let's build it using **two 74LS139 (Dual 2-to-4 Decoder)** and **four 74LS157 (Quad 2-to-1 MUX)**. This is getting too complex.

**Let's use a much simpler and more direct approach: Logic Gates.** This is more fundamental and easier to trace.

---

### **3. Revised Implementation Strategy: Logic Gates (Sum-of-Products)**

We will derive a logic equation for each of the four output bits (`ALU_Control_3`, `_2`, `_1`, `_0`) and implement them with standard 74-series gates.

#### **Step 3.1: Derive the Truth Table**

| ALUOp1 | ALUOp0 | Func3 | Func2 | Func1 | Func0 | ALU_Control_3 | ALU_Control_2 | ALU_Control_1 | ALU_Control_0 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 0 | 0 | X | X | X | X | 0 | 0 | 0 | 0 |
| 0 | 1 | X | X | X | X | 0 | 0 | 0 | 1 |
| 1 | 0 | F3 | F2 | F1 | F0 | F3 | F2 | F1 | F0 |
| 1 | 1 | X | X | X | X | 0 | 0 | 0 | 0 |

*(X = Don't Care, F3 = Func3, etc.)*

#### **Step 3.2: Derive Logic Equations**

We write an equation for each output bit. A '1' in the output column gives us a product term.

*   **`ALU_Control_3` (Most Significant Bit):**
    *   It is '1' only when `ALUOp=10` AND `Func3=1`.
    *   Equation: `ALU_Control_3 = (ALUOp1 AND NOT(ALUOp0) AND Func3)`

*   **`ALU_Control_2`:**
    *   Equation: `ALU_Control_2 = (ALUOp1 AND NOT(ALUOp0) AND Func2)`

*   **`ALU_Control_1`:**
    *   Equation: `ALU_Control_1 = (ALUOp1 AND NOT(ALUOp0) AND Func1)`

*   **`ALU_Control_0` (Least Significant Bit):**
    *   It is '1' in two cases:
        1.  When `ALUOp=01` (for SUBTRACT).
        2.  When `ALUOp=10` AND `Func0=1` (for R-Type).
    *   Equation: `ALU_Control_0 = (NOT(ALUOp1) AND ALUOp0) OR (ALUOp1 AND NOT(ALUOp0) AND Func0)`

### **5. Step-by-Step Wiring Guide (Proteus & Hardware)**

Let's build the circuit for the four output bits one by one.

#### **Common Signals:**
First, create the intermediate signals you'll need. You can do this with a **74LS04 (Inverter)**.
*   `NOT_ALUOp0` = Connect `ALUOp0` to a NOT gate input.
*   `NOT_ALUOp1` = Connect `ALUOp1` to a NOT gate input.

#### **Building `ALU_Control_3`, `_2`, and `_1`:**
The logic for these three bits is identical in form, just with a different `Func` bit input. We will use a **74LS08 (AND gate)** for each.

*   **For `ALU_Control_3`:**
    1.  Use a 3-input AND gate. You can build this by chaining two 2-input AND gates from a **74LS08**.
    2.  **Input 1:** `ALUOp1`
    3.  **Input 2:** `NOT_ALUOp0` (from the 74LS04)
    4.  **Input 3:** `Func3` (this is `D4` from your instruction register)
    5.  The output of this AND gate is `ALU_Control_3`.

*   **For `ALU_Control_2`:**
    1.  Repeat the exact same setup.
    2.  **Input 1:** `ALUOp1`
    3.  **Input 2:** `NOT_ALUOp0`
    4.  **Input 3:** `Func2` (this is `D3` from your instruction register)
    5.  The output is `ALU_Control_2`.

*   **For `ALU_Control_1`:**
    1.  Repeat the exact same setup.
    2.  **Input 1:** `ALUOp1`
    3.  **Input 2:** `NOT_ALUOp0`
    4.  **Input 3:** `Func1` (this is `D2` from your instruction register)
    5.  The output is `ALU_Control_1`.

#### **Building `ALU_Control_0`:**
This one is more complex as it requires an OR gate.

1.  **Create the first product term:** `(NOT(ALUOp1) AND ALUOp0)`
    *   Use one AND gate on a **74LS08**.
    *   **Input 1:** `NOT_ALUOp1` (from the 74LS04)
    *   **Input 2:** `ALUOp0`
    *   Let's call the output `Term1`.

2.  **Create the second product term:** `(ALUOp1 AND NOT(ALUOp0) AND Func0)`
    *   Use a 3-input AND gate (two 2-input AND gates chained).
    *   **Input 1:** `ALUOp1`
    *   **Input 2:** `NOT_ALUOp0`
    *   **Input 3:** `Func0` (this is `D1` from your instruction register)
    *   Let's call the output `Term2`.

3.  **Combine the terms:**
    *   Use one OR gate on a **74LS32**.
    *   **Input 1:** `Term1`
    *   **Input 2:** `Term2`
    *   The output of this OR gate is `ALU_Control_0`.

---

### **6. Final Assembly and Verification**

1.  **Place the ICs:** Put one 74LS04, two 74LS08s, and one 74LS32 on your Proteus workspace or breadboard.
2.  **Power Up:** Connect `Vcc` (pin 14) and `GND` (pin 7) for all ICs.
3.  **Route Inputs:**
    *   Connect `ALUOp1` and `ALUOp0` from your Main Control Unit to the inputs of the 74LS04 and 74LS08s as described.
    *   Connect `Func[3:0]` (bits `D4, D3, D2, D1`) from your Instruction Register to the specified AND gate inputs.
4.  **Route Outputs:** The outputs of the AND gates (for bits 3, 2, 1) and the final OR gate (for bit 0) are your `ALU_Control[3:0]` signals. Connect these directly to the 4-bit operation code input of your ALU.
5.  **Test:**
    *   **Test Case 1 (LOAD):** Force `ALUOp` to `00`. The outputs should be `0000`.
    *   **Test Case 2 (BRANCH):** Force `ALUOp` to `01`. The outputs should be `0001`.
    *   **Test Case 3 (R-Type AND):** Force `ALUOp` to `10` and `Func` to `0111` (for AND, from your table). The outputs should become `0111`.
    *   **Test Case 4 (R-Type XOR):** Force `ALUOp` to `10` and `Func` to `1001`. The outputs should become `1001`.

This gate-level implementation is robust, easy to debug with an oscilloscope or logic analyzer, and directly follows the logical requirements of your CPU design.
